码而优则仕的3个底层认知

Posted on 2022-03-20 Edited on 2022-03-31

“学而优则仕”的传统文化渗透在各行各业。

那么”码而优则仕”则是这种文化的延伸，自然而然的结果。

当然很多程序员对做管理不感兴趣，想很纯粹的研究技术，成为一名技术专家。人各有志，但客观讲，技术专家与技术管理相比较，成为技术专家比技术管理难度大得多，只有做到在一个区域，一个行业屈指可数的顶尖专家，才有更加广阔的前景。尤其在国内的环境大多数都是业务型公司，技术本身没有直接价值，没有太高的壁垒，有太多能取代你位置的人。

当然，技术管理也有难度，尤其对于程序员群体。但只要做到一个团队内的最佳，则拿到了敲门砖，随时准备幸运之神的降临。跨越技术纯粹性的舒适性，技术管理之路就开启了。

当被首次提拔为技术管理时，会本能地做些什么呢？

“我要让他们这样写代码”“以后这种事的流程这么走”“以后每周我们进行一次code review”“大家high起来”···

这些很笼统，作为一名非管理专业且刚入门的管理新手，如何华丽转身？

唯一驱动自我成长的帮助，就是寻找各种管理资料。

各门各派的管理学说：

1、 管理没那么复杂，只要做好四件事：把管、理、人、事组成四象限

2、 管理第一件事就是招聘

招聘第一原则：宁缺毋滥

基本要求：

  1. coding能力
  2. 对技术的热情
  3. 能简明扼要地沟通
  4. 积极乐观
  5. 对团队目标的认同

3、 打造团队是日常工作首当其冲的任务

作为团队Leader，都要走在团队前面，为团队规划方向、定义技术路线和架构模型、建立与之匹配的团队组织结构、团队骨干和backup

4、 Leader的基本要求

不仅基本功要扎实，还得有架构能力、抽象能力、设计能力，包括前瞻性技术规划的能力

5、 设计规范与制定流程

如开发规范：命名规范、代码模板、maven规范、comment规范、统一API规范、异常规范、分支规范、日志规范

制定流程：需求管理、架构评审、代码评审、发布评审

6、 技术管理能力

技术视野和技术格局，以及诸如交付能力、运行保障能力、带团队能力等等，真正落在具体工作场景中的核心技术能力，团队规划、绩效考核、团队建设

7、 研发Leader的职责

  1. 团队任务管理：开发工作量评估、开发任务分配
  2. 团队生产质量提升：代码审核、开发风险识别/报告/协调解决
  3. 团队生产力提升：代码模板研发与推广、最佳实践规范总结与推广、自动化研发生产工具研发与推广
  4. 团队专业力提升：招聘面试、新人指导、领导复盘总结改进

看了上面的各个流派，是不是蒙圈了，没法干了，不知道到底要干什么？

后面我去查找了管理的学术资料，管理的定义很简单：通过他人完成工作。

明确不要再务实了，天天跟下属抢活干。

并参加了一些管理培训，管理的4个任务

  1. 达成目标：承接上级意图，带领团队追求卓越成果
  2. 带好团队：明确方向，建立运作规则，奖罚公正
  3. 发展自我：提升岗位能力，面向未来追求进步
  4. 传承文化：身体力行，树立榜样

根据所学，把管理定义进行一下扩句：

  1. 让他人完成工作
  2. 让他人高效完成工作
  3. 让他人高效高质量完成工作
  4. 让他人愉快高效高质量完成工作
  5. 让他人主动愉快高效高质量完成工作

虽然流派众多，干法万千，然而这些都是别人的，管理是艺术，终究需要结合自身特点，汇融众派集一身，形成自己的管理特色。

在如此众多纷繁的方法论面前，怎么才能寻找到自己的特色，刘润说在快速变化的环境里面，我们要寻找底层逻辑，在乔新亮的CTO复盘中，发现了三个底层认知。

第一个：组织调整到位

乔老师讲的是职能型团队与产品型团队。

对于一线管理者，组织调整，我想更多的是人才梯队。一个团队不可能都是强者，强强联合不如强弱组合。

第二个：加强协同效率

为什么要搞一堆开发规范，开会章程？不也是为了书同文，车同轨，让协作更流畅。

为什么要建立坦诚透明的文化？也是为了减少信息差，上下同心，避免相互推诿。

第三个：激活团队活力

为什么要团建？公司是钱多请客吃饭？

为什么搞考评淘汰，晋升制度？

在招聘时，都要考察价值观？

这些手段都是为了激活团队活力。

当团队中越来越多的自律、自驱的同路人。先管，向管理要效益；然后慢慢不管，团队自律、自驱，管理达到无为状态。

这三个底层认知是所有方法论的抽象，而在落地做事时，又得具象化。

然而我想最根本的一条管理解法：业务持续高速发展，就如同个人一样：“何以解优，唯有暴富”。

当公司业务在高速发展，野蛮扩张时，谁会关心管理细则，业务推着每个人前进，心中有成就感，财富也源源而来。心满意，钱到位。员工全火力猛干，老板也没空来看考勤记录，人员成本管控。

但高速发展不会是常态，也就是管理不能永远缺席。管理是公司风险控制的重要保障。

老子曾说：太上，不知有之；其次，亲而誉之；其次，畏之；其次，侮之。信不足焉，有不信焉。所以管理终极目标是不管。管理的目的是为了将不那么自我驱动的人，变得更主动、更积极，而不是当个监工，越管越严。

而且管理者的职责不是保证每个人的成功，而是保证组织以及留 在组织中成员成功。如果管理者只做好人，最后大家短期开心，长期组织死掉了，最终把所有人都害了。
